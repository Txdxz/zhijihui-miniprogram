# 订阅消息配置指南

## 概述

智机惠小程序使用微信订阅消息功能，在以下关键业务节点通知管理员：

1. **待核验通知** - 用户提交手机号后，通知管理员核验资格
2. **验证码待处理** - 用户提交验证码后，通知管理员及时办理合约
3. **新验证码通知** - 用户重新提交验证码后，通知管理员处理新验证码

---

## 一、申请订阅消息模板

### 1. 登录微信公众平台

访问 [微信公众平台](https://mp.weixin.qq.com/)，使用小程序管理员账号登录。

### 2. 进入订阅消息管理

路径：功能 → 订阅消息 → 公共模板库

### 3. 搜索并选用模板

需要申请 3 个模板：

#### 模板1：业务处理通知（用于待核验通知）

在模板库搜索关键词：**业务处理** 或 **新任务**

推荐选择类似以下字段的模板：
- 事情名称：新客户提交申请
- 客户信息：手机号
- 门店信息
- 提交时间

记录模板 ID，记为 `TEMPLATE_ID_VERIFY_PENDING`

#### 模板2：验证码提醒（用于验证码待处理）

在模板库搜索关键词：**验证码** 或 **提醒**

推荐选择类似以下字段的模板：
- 事情名称：验证码待处理
- 客户信息：手机号
- 验证码内容
- 门店信息
- 时间

记录模板 ID，记为 `TEMPLATE_ID_SMS_CODE_PENDING`

#### 模板3：新验证码通知

可复用模板2，或申请新模板。

记录模板 ID，记为 `TEMPLATE_ID_NEW_SMS_CODE`

## 二、配置云函数

### 1. 修改云函数代码

打开 `cloudfunctions/sendSubscribeMessage/index.js`，找到 `TEMPLATE_IDS` 配置：

```javascript
const TEMPLATE_IDS = {
  // 将下面的占位符替换为实际的模板 ID
  VERIFY_PENDING: 'TEMPLATE_ID_VERIFY_PENDING',    // 替换为模板1的ID
  SMS_CODE_PENDING: 'TEMPLATE_ID_SMS_CODE_PENDING', // 替换为模板2的ID
  NEW_SMS_CODE: 'TEMPLATE_ID_NEW_SMS_CODE'         // 替换为模板3的ID
};
```

### 2. 修改示例

假设你在公众平台申请到的模板 ID 分别是：
- 待核验通知：`abc123xxx`
- 验证码待处理：`def456xxx`
- 新验证码：`ghi789xxx`

则配置修改为：

```javascript
const TEMPLATE_IDS = {
  VERIFY_PENDING: 'abc123xxx',
  SMS_CODE_PENDING: 'def456xxx',
  NEW_SMS_CODE: 'ghi789xxx'
};
```

### 3. 上传云函数

```
右键 cloudfunctions/sendSubscribeMessage → 上传并部署：云端安装依赖
右键 cloudfunctions/portalBiz → 上传并部署：云端安装依赖
```

## 三、管理员订阅通知

### 1. 进入管理后台

管理员登录小程序，进入「管理员」tab

### 2. 开启通知

找到「业务通知」卡片，点击「开启通知」按钮

### 3. 授权订阅

微信会弹出订阅授权弹窗，勾选需要订阅的通知类型，点击「允许」

---

## 四、注意事项

1. **订阅有效期**：用户每次授权只能发送一次订阅消息。管理员需要定期重新订阅。

2. **模板字数限制**：模板字段有字数限制，超出会被截断。

3. **跳转页面**：点击通知会跳转到管理后台首页，建议使用 `miniprogramState: 'formal'` 确保正式版正常跳转。

4. **测试环境**：开发版和体验版需要在 `sendSubscribeMessage` 函数中修改 `miniprogramState` 参数。

---

## 五、故障排查

### 问题1：点击订阅没有反应

- 检查是否已申请订阅消息模板
- 检查模板 ID 是否正确配置
- 查看开发者工具控制台错误信息

### 问题2：收不到通知

- 确认管理员已点击授权
- 检查云函数日志是否有错误
- 确认模板字段名称与配置一致

### 问题3：通知内容不完整

- 检查模板字段配置是否正确
- 确认传递的数据不为空
