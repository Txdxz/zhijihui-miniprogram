# 智机惠小程序 - 数据字典

> **文档版本**：v1.0
> **更新日期**：2026-06-17
> **面向对象**：第三方开发团队
> **数据库类型**：微信云开发 CloudBase（文档型数据库，兼容 MongoDB 语法）

---

## 一、文档说明

### 1.1 文档目的

本文档为微信小程序「智机惠」的数据库集合数据字典，面向第三方开发团队，用于：

- 了解系统数据库整体结构
- 掌握每个集合的字段定义、类型、约束与枚举值
- 明确索引设计与权限规则
- 作为二次开发、数据迁移、运维排查的依据

### 1.2 通用约定

| 约定项 | 说明 |
|--------|------|
| 数据库引擎 | 微信云开发数据库（基于 MongoDB） |
| 主键 | 系统自动生成 `_id` 字段（string），业务主键见各集合说明 |
| 日期类型 | 统一使用 ISO 8601 字符串（UTC），例如 `2026-06-17T08:00:00.000Z` |
| 时间戳类型 | 毫秒级数字时间戳，例如 `1718611200000` |
| 布尔值 | 使用 number 表示，`1` 为真，`0` 为假 |
| 空值约定 | 字符串字段未赋值时统一为 `''`（空字符串），日期字段未赋值时为 `''` 或 `null` |
| 命名规范 | 字段名采用小驼峰（camelCase），集合名采用下划线（snake_case） |
| ID 生成 | 业务 ID 由云函数生成，带前缀（如 `C`、`CP`、`R`、`store_`） |

## 二、权限规则（全局统一）

所有集合统一采用自定义安全规则，**禁止客户端直接读写**，所有数据访问必须通过云函数进行。

### 2.1 权限配置

```json
{
  "read": false,
  "write": false
}
```

### 2.2 配置说明

| 规则字段 | 取值 | 说明 |
|----------|------|------|
| `read` | `false` | 禁止客户端直接读取，所有数据访问通过云函数进行 |
| `write` | `false` | 禁止客户端直接写入，所有数据操作通过云函数进行 |

### 2.3 配置方式

1. 进入云开发控制台 → 数据库 → 选择对应集合
2. 点击「权限设置」标签
3. 选择「自定义安全规则」
4. 粘贴上述 JSON 规则并保存
5. **10 个集合均需按此规则配置**

### 4.2 portal_roles（角色权限表）

**用途**：存储用户的角色权限信息，支持多角色体系，通过 scopeType/scopeId 控制权限作用范围。

#### 字段列表

| 字段名 | 类型 | 必填 | 默认值 | 说明 |

| `name` | string | ❌ | `''` | 角色名称/用户姓名 |
| `phone` | string | ❌ | `''` | 联系手机号 |
| `scopeType` | string | ✅ | - | 权限作用范围类型 |
| `scopeId` | string | ❌ | `''` | 权限作用范围 ID（如 storeId） |
| `scopeName` | string | ❌ | `''` | 权限作用范围名称 |
| `permissions` | array | ❌ | `[]` | 权限列表（权限键字符串数组） |
| `status` | number | ✅ | `1` | 角色状态 |
| `boundBy` | string | ❌ | - | 角色绑定方式 |
| `createdAt` | date | ✅ | - | 创建时间 |
| `updatedAt` | date | ✅ | - | 更新时间 |

#### 枚举值说明

**`roleKey` 字段**

| 值 | 含义 |
|----|------|
| `customer` | 普通客户 |

| 值 | 含义 | 说明 |

### 4.4 contracts（合约表）

**用途**：存储客户业务办理的全流程记录，包含从申请提交到合约完成的完整状态流转。

#### 字段列表

| 字段名 | 类型 | 必填 | 默认值 | 说明 |
|--------|------|------|--------|------|
| `contractId` | string | ✅ | - | 合约唯一标识（前缀 `C`） |
| `openId` | string | ✅ | - | 客户 openId |
| `phone` | string | ✅ | - | 客户手机号 |
| `name` | string | ❌ | `''` | 收货人姓名 |
| `address` | string | ❌ | `''` | 收货地址 |
| `storeId` | string | ✅ | - | 办理门店 ID |
| `storeName` | string | ❌ | `''` | 办理门店名称（冗余字段） |
| `status` | number | ✅ | `1` | 合约状态 |
| `smsCode` | string | ❌ | `''` | 短信验证码 |
| `smsCodeRejectedAt` | date | ❌ | `null` | 验证码驳回时间 |
| `smsCodeRejectReason` | string | ❌ | `''` | 验证码驳回原因 |
| `trackingNo` | string | ❌ | `''` | 物流单号 |
| `logistics` | array | ❌ | `[]` | 物流轨迹信息列表 |
| `shippedAt` | date | ❌ | `''` | 发货时间 |
| `contractOkAt` | date | ❌ | `''` | 合约办理完成时间 |
| `signedAt` | date | ❌ | `''` | 签收时间 |
| `createdAt` | date | ✅ | - | 创建时间 |
| `updatedAt` | date | ✅ | - | 更新时间 |

#### 枚举值说明

**`status` 字段（合约状态机）**

| 值 | 常量名 | 含义 | 说明 |
|----|--------|------|------|
| `0` | `REJECTED` | 拒绝 | 门店信息核验未通过 |
| `1` | `WAIT_VERIFY` | 待核验 | 用户已提交手机号与门店，等待核验 |
| `2` | `QUALIFIED` | 核验通过 | 门店信息核验通过，待提交收货信息 |
| `3` | `WAIT_SMSCODE` | 待验证码 | 收货信息已提交，等待用户输入短信验证码 |
| `4` | `CONTRACTING` | 办理中 | 验证码校验通过，合约办理中 |
| `5` | `CONTRACT_OK` | 合约已办 | 合约办理完成，待发货 |
| `6` | `SHIPPED` | 已发货 | 已发货，等待签收 |
| `7` | `SIGNED` | 已完成 | 用户已签收，流程结束 |
| `8` | `SMS_CODE_REJECTED` | 验证码被驳回 | 验证码错误，需用户重新输入 |

**状态流转图**

```
[1 待核验] ──拒绝──> [0 拒绝]
     │
     └─核验通过─> [2 核验通过] ──提交收货信息──> [3 待验证码]
                                                  │
                                  ┌──验证码错误──> [8 验证码被驳回]
                                  │               │
                                  │               └─重新输入─> [3 待验证码]
                                  │
                                  └─验证码正确─> [4 办理中] ──> [5 合约已办]
                                                                   │
                                                                   └─发货─> [6 已发货] ──签收──> [7 已完成]
```

| `openId` | 单字段 | ❌ |
| `status` | 单字段 | ❌ |
| `createdAt` | 单字段（降序） | ❌ |

## 五、集合关系说明

### 5.1 核心实体关系

```
portal_users (1) ──────< (N) portal_roles
     │                        │
     │                        │ scopeId → stores.storeId
     │
     │ currentContractId
```

## 六、开发注意事项

### 6.1 数据访问规范

1. **禁止客户端直连数据库**：所有数据访问必须通过云函数，集合权限已统一设置为 `read:false, write:false`
2. **日期统一使用 ISO 字符串**：写入数据库时使用 `new Date().toISOString()` 生成
3. **业务 ID 由云函数生成**：避免客户端伪造，ID 带业务前缀便于识别

### 6.2 数据一致性

1. **冗余字段**：`contracts.storeName`、`coupons.storeName` 为冗余字段，门店名称变更时需同步更新
2. **状态变更**：合约状态变更需同步更新 `portal_users.currentContractId`
3. **软删除**：`coupon_rules`、`stores` 等采用软删除（status=0），不物理删除

### 6.3 索引维护

1. **唯一索引**：`portal_users.openId`、`stores.storeId`、`contracts.contractId`、`coupons.couponId` 为唯一索引，写入重复数据会报错
2. **索引创建**：可通过 `initDatabase` 云函数自动创建，或在云开发控制台手动添加
3. **复合查询**：如需复合查询（如 `openId + status`），可按需新增复合索引

### 6.4 敏感数据处理

1. **手机号**：`phone` 字段为敏感信息，日志输出时需脱敏
2. **验证码**：`sms_verifications.code` 为敏感信息，校验后应及时更新状态
3. **openId**：`openId` 为用户唯一标识，不可对外暴露
