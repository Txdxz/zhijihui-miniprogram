## 1. 系统概述

"智机惠"是一款面向中国移动河北用户的合约办理与权益核销微信小程序。系统通过与中国移动【河北移动权益优选】小程序对接，实现"合约办理 → 代金券发放 → 权益激活 → 门店核销"的完整闭环。

系统包含四类核心角色：

| 角色 | 标识 | 主要职责 |
|------|------|----------|
| 客户 | `customer` | 办理合约、领取权益、到店核销 |
| 门店店员 | `store_clerk` | 门店核销代金券 |
| 门店店长 | `store_owner` | 门店核销 + 门店管理 |
| 管理员 | `admin` / `super_admin` | 合约审核、办理、发货、系统管理 |

### 2.3 中国移动权益对接流程

**流程说明：** 智机惠合约完成后，代金券需在中国移动【河北移动权益优选】小程序激活，激活后回传至智机惠供门店核销。

```
┌─────────────────────────────────────────────────────────────────┐
│            智机惠合约完成（status=5 合约已办）                     │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
                ┌─────────────────────────────┐
                │  系统自动生成代金券            │
                │  coupon.status = 0 待激活     │
                └─────────────────────────────┘
                              │
                              ▼
                ┌─────────────────────────────┐
                │  用户跳转至中国移动小程序      │
                │  【河北移动权益优选】          │
                └─────────────────────────────┘
                              │
                              ▼
                ┌─────────────────────────────┐
                │  用户点击"去使用"领取权益      │
                └─────────────────────────────┘
                              │
                              ▼
                ┌─────────────────────────────┐
                │  移动平台回调智机惠接口        │
                │  activateCoupon              │
                └─────────────────────────────┘
                              │
                              ▼
                ┌─────────────────────────────┐
                │  代金券激活                    │
                │  coupon.status = 1 可使用     │
                │  ★ 30天有效期开始计算         │
                └─────────────────────────────┘
                              │
                              ▼
                ┌─────────────────────────────┐
                │  用户回到智机惠小程序          │
                │  查看卡包 pages/coupons       │
                └─────────────────────────────┘
                              │
                              ▼
                ┌─────────────────────────────┐
                │  用户到店出示核销码            │
                │  门店扫码核销                 │
                └─────────────────────────────┘
                              │
                              ▼
                ┌─────────────────────────────┐
                │  代金券状态变更                │
                │  coupon.status = 2 已核销     │
                └─────────────────────────────┘
                              │
                              ▼
                ┌─────────────────────────────┐
                │  智机惠回传核销状态给中国移动   │
                │  （异步通知）                  │
                └─────────────────────────────┘
```

**对接要点：**
- `activateCoupon` 接口需校验移动平台签名，防止伪造回调
- 30 天有效期自激活时刻起算，到期自动变更为 `status=3 已过期`
- 核销状态回传为异步通知，需设计重试机制

**关键接口：**
- `activateCoupon`：移动平台回调激活代金券
- 核销状态回传接口（向移动平台推送）

```

**核销码规则：**
- 6 位数字
- 有效期 3 分钟，过期需重新生成
- 二维码内容为核销码本身（或带签名的 URL）

**关键接口：**
- `generateVerifyCode`：生成核销码
- `storeVerifyCoupon`：查询券信息（核销前校验）
- `storeConfirmVerify`：确认核销

## 3. 角色权限矩阵

下表定义了系统中各角色可执行的功能操作。`✓` 表示允许，`-` 表示禁止。

| 功能 | customer | store_clerk | store_owner | admin | super_admin |
|------|:--------:|:-----------:|:-----------:|:-----:|:-----------:|

                    ┌─────────────────────────────────────────────────────┐

                   ▼                                                      │
              ┌──────────┐                                                │
       ┌─────→│ 1 待核验 │                                                │
       │      └────┬─────┘                                                │
       │           │                                                      │
       │           │ 管理员核验通过                                        │
       │           ▼                                                      │
       │      ┌──────────┐                                                │
       │      │ 2 核验   │                                                │
       │      │  通过    │                                                │

       │      │   码     │  驳回   │                  │                  │
       │      └────┬─────┘         └────────┬─────────┘                  │
       │           │                        │                            │
       │           │ 验证码通过              │ 客户重新输入                │
       │           ▼                        └────────────────────────────┘
       │      ┌──────────┐
       │      │ 4 办理中 │
       │      └────┬─────┘
       │           │
       │           │ 管理员办理合约完成
       │           │ ★ 自动生成代金券（status=0 待激活）
       │           ▼
       │      ┌──────────┐
       │      │ 5 合约   │
       │      │  已办    │
       │      └────┬─────┘
       │           │
       │           │ 管理员发货
       │           ▼

       │           │

       │      │ 7 已完成 │  （终态）
       │      └──────────┘
       │

```

**状态定义：**

| 状态值 | 状态名 | 说明 | 触发动作 |
|:------:|--------|------|----------|
| 0 | 拒绝 | 管理员核验未通过 | 管理员拒绝核验 |

| 2 | 核验通过 | 管理员核验通过 | 管理员核验通过 |
| 3 | 待验证码 | 等待客户输入短信验证码 | 客户 submitOrderInfo |
| 4 | 办理中 | 管理员正在办理合约 | 客户 submitSmsCode 通过 |
| 5 | 合约已办 | 合约办理完成，自动生成代金券 | 管理员办理完成 |
| 6 | 已发货 | 管理员已发货 | 管理员发货 |
| 7 | 已完成 | 客户签收，流程终结 | 客户签收 |
| 8 | 验证码被驳回 | 验证码错误，需重新输入 | 管理员驳回验证码 |

### 4.2 代金券状态机

代金券状态机描述代金券从生成到核销（或过期）的状态流转。

```
   ┌────────────────┐
   │ 0 待激活        │  ← 合约状态=5 时系统自动生成
   │ (status=0)     │
   └───────┬────────┘
           │
           │ 中国移动回调 activateCoupon
           │ 30 天有效期开始计算
           ▼
   ┌────────────────┐
   │ 1 可使用        │
   │ (status=1)     │
   └───────┬────────┘
           │
           ├──────────────────────────────┐
           │                              │
           │ 门店核销                      │ 超过 30 天有效期
           │ storeConfirmVerify           │ 定时任务扫描
           ▼                              ▼
   ┌────────────────┐              ┌────────────────┐
   │ 2 已核销        │              │ 3 已过期        │
   │ (status=2)     │              │ (status=3)     │
   │ （终态）        │              │ （终态）        │
   └────────────────┘              └────────────────┘
           │
           │ 异步回传移动核销状态
           ▼
      [通知中国移动]
```

**状态定义：**

| 状态值 | 状态名 | 说明 | 触发动作 |
|:------:|--------|------|----------|
| 0 | 待激活 | 合约完成后生成，等待移动平台激活 | 系统自动生成 |
| 1 | 可使用 | 已激活，30 天有效期内可核销 | 移动平台回调 activateCoupon |
| 2 | 已核销 | 门店已完成核销 | storeConfirmVerify |
| 3 | 已过期 | 超过 30 天有效期 | 定时任务自动变更 |

**关键规则：**
- 30 天有效期自 `status=1` 激活时刻起算
- `status=2` 和 `status=3` 均为终态，不可逆
- 核销后需异步回传移动平台核销状态

## 6. 异常处理与边界说明

### 6.1 登录与角色分流异常

| 异常场景 | 处理策略 |
|----------|----------|
| `login` 云函数失败 | 提示网络异常，提供重试按钮 |
| `resolveLaunchContext` 超时 | 降级为普通客户身份进入首页 |
| 用户拒绝手机号授权 | 降级为普通客户，不绑定角色 |
| 邀请记录手机号与授权手机号不匹配 | 提示"手机号与邀请不匹配"，不执行绑定 |

### 6.2 合约办理异常

| 异常场景 | 处理策略 |
|----------|----------|
| 验证码被驳回（status=8） | 允许客户重新输入，回到 status=3 |
| 管理员核验拒绝（status=0） | 客户可重新提交，回到 status=1 |
| 收货信息填写超时 | 保留 status=2，允许继续填写 |
| 合约办理中服务异常 | 管理员可手动回退至 status=3 |

### 6.3 代金券与核销异常

| 异常场景 | 处理策略 |
|----------|----------|
| 移动平台 activateCoupon 回调签名校验失败 | 拒绝激活，记录日志告警 |
| 核销码过期（超过 3 分钟） | 提示用户重新生成核销码 |
| 重复核销（券已为 status=2） | 拒绝核销，提示"券已核销" |
| 券已过期（status=3） | 拒绝核销，提示"券已过期" |
| 门店归属校验失败 | 拒绝核销，提示"非本门店代金券" |
| 移动平台核销状态回传失败 | 异步重试，最多 3 次，失败告警 |

### 6.4 邀请绑定异常

| 异常场景 | 处理策略 |
|----------|----------|
| 邀请已过期 | 提示"邀请已失效"，联系超级管理员重新发起 |
| 邀请已被绑定（boundOpenId 已存在） | 提示"邀请已被使用"，拒绝重复绑定 |
| 手机号解密失败 | 提示重新授权，记录错误日志 |
